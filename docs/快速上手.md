# 快速上手

从安装到完成第一个 MicroPython 程序刷入，涵盖 pyrite-cli 的安装、核心概念、基本命令和配置方式。适合首次接触 pyrite-cli 的用户。

---

## 1. 安装与环境

### 1.1 标准安装

```bash
pip install pyrite-cli
```

核心依赖自动安装：`typer`、`pyserial`、`requests`、`tqdm`、`mpy-cross`、`mpremote`、`libcst`。

### 1.2 开发安装

```bash
git clone <仓库地址>
cd pyrite-cli
pip install -e .
```

可编辑模式下代码修改即时生效。

### 1.3 验证安装

```bash
pyrcli --help
pyrcli scan --version
```

## 2. 核心架构一览

pyrite-cli 的分层架构：

```
CLI 入口 (main.py)          ← 你敲的命令
    │
    ├── 传输层              ← 串口或 WebSocket
    │   └── transport/ 包：Transport ABC / SerialTransport；webrepl/ 包：WebREPLTransport / WebREPLMicroPython
    │
    ├── 刷入引擎 (flash/)   ← 原始 REPL 协议、文件传输
    │
    ├── 项目同步 (sync.py)   ← 哈希增量刷入、状态比对
    │
    ├── 开发会话与测试         ← project dev、设备端测试、traceback 映射
    │
    ├── 快照与回滚             ← 设备文件系统 snapshot、diff、restore
    │
    ├── 主机能力透传           ← tunnel kb / tunnel network
    │
    ├── 条件编译             ← libcst AST 转换，刷入前预处理
    │
    ├── 统一日志             ← 6 级日志、JSONL 记录、操作计时
    │
    └── JSON 输出            ← --format json 集成到脚本/CI
```

各模块职责：

| 模块 | 职责 |
|------|------|
| `main.py` | CLI 入口，所有命令定义 |
| `flash/` | 串口连接、原始 REPL 通信、文件刷入、设备文件浏览 |
| `sync.py` | 基于 SHA256 的增量同步、状态比对、批量拉取 |
| `transport/` | 传输层抽象与串口实现；WebREPL 在 `webrepl/` |
| `device_tests.py` | 上传并运行设备端测试，解析 `PYRITE_TEST` 结果帧 |
| `snapshot.py` | 本地保存设备文件快照，计算 diff 和恢复计划 |
| `tunnel.py` / `device/tunnel_scripts/` | 键盘事件和受限 HTTP(S) 请求透传 |
| `compiler.py` | mpy-cross 编译封装 |
| `preprocessor.py` | 条件编译宏预处理 |

> 深入理解各模块 → [设备刷入与项目同步](设备刷入与项目同步.md)

## 3. 命令速览

### 3.1 设备发现

```bash
pyrcli scan                # 扫描串口设备
pyrcli scan -a             # 显示所有设备（含无 VID/PID）
pyrcli debug board-info COM3     # 获取设备详细信息（固件版本、CPU、Flash）
```

### 3.2 交互式 REPL

```bash
pyrcli repl COM3           # 交互式 MicroPython REPL，可在其中执行代码
# 在 REPL 中输入：import machine; print(machine.freq())
pyrcli reset COM3          # 软重启设备
```

### 3.3 文件操作

```bash
pyrcli fs ls COM3 /              # 列出目录
pyrcli fs ls COM3 /lib -r        # 递归列出
pyrcli fs ls COM3 / --sort size  # 按大小排序
pyrcli fs ls COM3 / -p           # 分页显示
pyrcli fs cat COM3 /main.py      # 查看文件内容
pyrcli fs put COM3 local.py /remote.py  # 上传文件
pyrcli fs get COM3 /remote.py backup.py # 下载文件
pyrcli fs rm COM3 /remote.py     # 删除文件
```

### 3.4 刷入

```bash
pyrcli flash COM3 main.py /main.py           # 单文件
pyrcli flash-program COM3 src/ /app          # 整个目录
pyrcli flash-program COM3 . / --manifest manifest.py  # manifest 过滤
```

### 3.5 包安装

```bash
pyrcli pkg install COM3 aioble --target /lib --dry-run  # 查看 mpremote mip 安装计划
pyrcli pkg install COM3 aioble --target /lib            # 通过 mpremote mip install 安装
pyrcli pkg cache aioble --version latest --dry-run      # 规划本地缓存目录
pyrcli pkg install-offline COM3 .pyrite/pkg-cache/aioble
```

### 3.6 项目管理

```bash
pyrcli project new my-project                # 交互式创建项目
pyrcli project new . --platform COM3         # 串口自动检测创建
pyrcli project hash .                        # 计算 SHA256
pyrcli project flash COM3 . /                # 增量刷入
pyrcli project status COM3 . /               # 查看差异
pyrcli project pull COM3 . /                 # 从设备拉取
```

### 3.7 开发会话与设备端测试

```bash
pyrcli test COM3 test_device/ --timeout 15
pyrcli test COM3 test_device/ --keep-files
pyrcli project dev COM3 . /app --lens
pyrcli project dev COM3 . /app --test-on-save=all --test-path test_device/
pyrcli project dev COM3 . /app --once --no-repl --test-on-save=all
```

`pyrcli test` 会把本地测试文件上传到设备端临时目录执行，并只解析 `PYRITE_TEST` 前缀结果帧；测试里的普通 `print()` 会作为设备普通输出显示。`project dev --test-on-save` 可在保存并刷入成功后自动运行设备端测试，`--lens` 会把 traceback 展开到本地源码上下文。

### 3.8 快照与恢复

```bash
pyrcli snapshot save COM3 before-change --remote-path /app
pyrcli snapshot list
pyrcli snapshot diff COM3 before-change --remote-path /app
pyrcli snapshot restore COM3 before-change
pyrcli snapshot restore COM3 before-change --apply --yes
pyrcli project flash COM3 . /app --snapshot-before before-flash
```

`snapshot restore` 默认只输出 dry-run 计划；真正写回设备必须显式加 `--apply`，无人值守时再加 `--yes`。当前对比阶段会下载设备文件并在 host 侧计算 SHA256，避免依赖旧固件上的 `hashlib.sha256`；未指定 `--remote-path` 时，恢复命令默认只扫描快照文件的共同父目录，避免误把无关设备路径纳入删除计划。

### 3.9 主机能力透传

```bash
pyrcli tunnel kb COM3
pyrcli tunnel network COM3 --allow example.com
pyrcli tunnel network COM3 --allow 127.0.0.1 --allow-private
pyrcli tunnel network COM3 --ws ws://192.168.4.1:8266/ --allow example.com --allow-webrepl
```

`tunnel kb` 把上位机键盘事件透传给设备端 helper。`tunnel network` 让设备通过上位机发起受限 HTTP(S) 请求，必须显式配置 `--allow`；访问 localhost、私有地址或链路本地地址还需要 `--allow-private`。WebREPL 连接下启动 network tunnel 需要额外加 `--allow-webrepl`。

### 3.10 挂载

```bash
pyrcli mount COM3             # PC 文件管理器访问设备文件系统
pyrcli remount COM3 .         # 设备端通过 /remote 访问当前上位机目录
pyrcli remount COM3 src/ --unsafe-links
```

`mount` 是 PC 侧 WebDAV 桥，方向是“上位机访问设备文件系统”。`remount` 是反向挂载，委托 `mpremote mount`，方向是“设备访问上位机目录”。

### 3.11 GPIO 监控

```bash
pyrcli monitor COM3 --pins 0,2,4,5 --interval 0.2 --count 20
pyrcli monitor COM3 --pins 0,2 --format json --count 5
pyrcli monitor COM3 --pins 0,2,4 --edge changed --duration 10
```

`monitor` 只以 `machine.Pin(pin, machine.Pin.IN)` 读取输入状态，不会设置输出模式或上下拉。未指定 `--pins` 时会保守探测常见 GPIO。

### 3.12 WebREPL 连接

所有设备命令均支持通过 WebSocket 连接（而非串口）：

```bash
pyrcli debug board-info COM3 --ws ws://192.168.4.1:8266/ --password mypass
pyrcli flash COM3 main.py /main.py --ws ws://esp32.local:8266
```

密码解析链：`--password` 参数 → `PYRITE_WEBREPL_PASSWORD` 环境变量 → 交互输入。

### 3.13 串口占用处理

当串口打开失败且错误看起来像“权限不足 / 已占用 / resource busy”时，交互式终端下会询问是否扫描并解除占用。Windows 优先使用 Sysinternals `handle.exe` 精确查找句柄，找不到时退回命令行启发式扫描；Linux/macOS 使用 `lsof` 或 `fuser`。结束进程前会再次确认，非交互环境会跳过该流程。

## 4. 快速示例：Hello World

### 4.1 创建设备文件

```python
# hello.py
import time

while True:
    print("Hello from MicroPython!")
    time.sleep(1)
```

### 4.2 扫描串口

```bash
pyrcli scan
# 输出示例：
# COM3 - USB Serial Device (VID:PID 10C4:EA60)
```

### 4.3 刷入文件

```bash
pyrcli flash COM3 hello.py /hello.py
```

### 4.4 运行验证

```bash
pyrcli repl COM3
# 在 REPL 中输入：exec(open('/hello.py').read())
# → Hello from MicroPython!
# → Hello from MicroPython!
```

### 4.5 查看设备文件

```bash
pyrcli fs ls COM3 /
# → hello.py  31  F
pyrcli fs cat COM3 /hello.py
# → import time
# → ...
```

## 5. 配置文件

### 5.1 `.pyrite_config.json`

项目根目录创建，从当前目录向上逐级搜索，找到即停止。不创建也能正常使用（全默认值）。

```json
{
  "chunk_size": 4096,
  "download_threads": 4,
  "auto_compile": true,
  "verify": "size",
  "delta_flash": "auto",
  "precheck": "basic",
  "precheck_compat": "warn",
  "precheck_mp_version": "",
  "max_retries": 2,
  "baudrate": 921600
}
```

| 字段 | 默认值 | 说明 |
|------|--------|------|
| `chunk_size` | `4096` | 每次写入的最大数据量（字节） |
| `download_threads` | `4` | 存根下载并发线程数，范围 1–12 |
| `auto_compile` | `true` | 是否自动编译 `.py` → `.mpy` |
| `verify` | `"size"` | 刷入后校验模式：`off`=不校验、`size`=文件大小、`crc32`=大小+CRC32 |
| `delta_flash` | `"auto"` | 单文件增量刷入策略：`off`/`auto`/`on` |
| `precheck` | `"basic"` | 刷入前代码预检查：`off`/`basic`/`strict` |
| `precheck_compat` | `"warn"` | strict 兼容性问题处理：`warn`/`error`/`off` |
| `precheck_mp_version` | `""` | 可选目标 MicroPython 固件版本，如 `1.20.0` |
| `max_retries` | `2` | 校验失败/断线最大重试次数，`0`=不重试 |
| `baudrate` | `921600` | 默认串口波特率，可按板卡稳定性调整 |

如果使用 config 命令生成：

```bash
pyrcli config
```

### 5.2 `pyproject.toml` Board Tags 补充

在 `pyproject.toml` 中扩展设备识别标签映射，会与内置默认标签合并：

```toml
[tool.pyrite.board_tags]
ESP32_S3 = ["ESP32", "wifi"]
C3 = ["ESP32", "wifi", "ble"]
MY_BOARD = ["MY_BOARD", "wifi", "sdcard"]
```

这些标签会影响条件编译中的 feature 自动激活。详见 [条件编译实战](条件编译实战.md)。

### 5.3 配置加载顺序

```
PyriteConfig() 内置默认
    ↓ 向上逐级查找
.pyrite_config.json（覆盖字段）
    ↓ 向上逐级查找
pyproject.toml [tool.pyrite.board_tags]（合并标签）
```
