## 1. 板侧离线 upypi/mip：上位机拉取

目标：设备不需要联网。上位机负责解析、下载、缓存 MicroPython 包，再通过 `mpremote` 安装到设备。

已实现命令面：

```bash
pyrcli pkg install COM3 aioble
pyrcli pkg install COM3 github:org/repo@branch --target /lib
pyrcli pkg cache aioble --version latest --dry-run
pyrcli pkg install-offline COM3 .pyrite/pkg-cache/aioble
```

实现状态：

1. `pkg install` 构建并执行 `mpremote connect <port> mip install <package>`，支持 `--target`、`--mpremote` 和 `--dry-run`。
2. `pkg install-offline` 校验本地 `package.json` 或包目录，再委托 `mpremote mip install <local>`。
3. `pkg cache` 目前是缓存路径规划和本地 `package.json` 审计入口，不自行执行网络下载。
4. 本地 manifest 审计覆盖 `deps` 和 `urls`，格式错误会给出来源路径和缺失项。
5. `--dry-run` 不连接设备、不调用 subprocess。

仍需后续扩展：

- 真正下载并落盘 `.pyrite/pkg-cache/<name>/<version>/` 的完整包内容和依赖锁定信息。
- 对 `.mpy` 版本与设备固件不匹配的情况提供更完整的回退或提示。

## 2. monitor 命令：GPIO 监控

目标：提供一个只读监控命令，周期性读取板上 GPIO 引脚状态；如果设备无法枚举引脚，则允许用户显式指定引脚列表。

已实现命令面：

```bash
pyrcli monitor COM3
pyrcli monitor COM3 --pins 0,2,4,5
pyrcli monitor COM3 --interval 0.2 --format json --count 20
pyrcli monitor COM3 --pins 0,2 --edge changed --duration 10
```

实现状态：

1. 未指定 `--pins` 时，设备端探测保守 GPIO 候选列表；探测失败时提示显式传入 `--pins`。
2. 采样只使用 `machine.Pin(pin, machine.Pin.IN)`，不设置上下拉，不设置输出模式。
3. host 侧按间隔循环注入一次采样短脚本，实时输出 text 或 JSON Lines。
4. 支持 `--duration`、`--count`、`--edge changed`。
5. 纯逻辑测试覆盖引脚解析、JSON 输出、探测脚本、采样脚本和 CLI runner。

风险点：

- 不同板卡 GPIO 编号、保留引脚和启动绑带引脚差异较大，默认探测列表保持保守；生产使用建议显式传入 `--pins`。

## 3. 结构优化

目标：整理项目结构，减少单文件体积，并保持旧导入兼容。

已实现结构：

```text
cli/utils/
  flash/
    __init__.py
    core.py
    flash.py
    mp_scripts/
  transport/
    __init__.py
    base.py
    serial.py
    webrepl.py
  serial_transport.py
  webrepl_transport.py
```

兼容策略：

- `from cli.utils.flash import MicroPython` 继续可用。
- `_strip_repl_trailer`、`_colorize_repl_output`、`_compute_block_crc32` 等 helper 继续从 `cli.utils.flash` 导出。
- `from cli.utils.transport import Transport` 继续可用。
- `cli.utils.serial_transport` 和 `cli.utils.webrepl_transport` 保留为薄 wrapper。
- `pyproject.toml` 的 package data 已改为 `cli.utils.flash = ["mp_scripts/*.py"]`。
