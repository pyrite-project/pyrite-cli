# 插件开发指南

pyrite-cli 支持通过第三方插件注册自定义命令。本文涵盖插件架构、打包规范、三种安装方式和发布流程。

> 前置阅读：[快速上手](快速上手.md) — 了解 pyrite-cli 基本用法

---

## 1. 架构概述

插件系统基于 **setuptools entry points**（Python 标准打包机制）进行发现，无需额外的注册表或配置文件。

```
第三方插件包                        pyrite-cli 宿主
─────────────────────               ──────────────
                                      cli/main.py
  pyproject.toml                          │
  └── [project.entry-points.              │
        "pyrite.commands"]                 │
        ota = "pyrite_ota:app"            ▼
                               cli/plugin_manager.py
  pyrite_ota/                   ┌──────────────────────────┐
  ├── __init__.py  ──►app──►    │ discover_plugins()       │
  └── commands.py               │   ↓                      │
                                │ load_plugin(ep)          │
                                │   ↓                      │
                                │ main_app.add_typer()     │
                                └──────────────────────────┘
```

启动时通过 `importlib.metadata.entry_points(group="pyrite.commands")` 扫描所有已安装包中匹配的 entry point。

## 2. 插件结构

### 2.1 目录布局

每个插件是一个标准 Python 包：

```
my_plugin/
├── __init__.py          # 导出 app: typer.Typer 实例
├── commands.py          # 命令实现
└── pyproject.toml       # 包元数据 + entry point
```

### 2.2 最小 `__init__.py`

```python
import typer

app = typer.Typer(help="我的插件命令")

__plugin_name__ = "my-plugin"
__plugin_version__ = "0.1.0"
```

### 2.3 最小 `pyproject.toml`

```toml
[project]
name = "pyrite-my-plugin"
version = "0.1.0"
dependencies = ["pyrite-cli>=0.1.0"]

[project.entry-points."pyrite.commands"]
my-plugin = "my_plugin:app"
```

### 2.4 命令实现（`commands.py`）

```python
from . import app

@app.command()
def hello(
    name: str = typer.Argument("world", help="要问候的名字"),
):
    """向某人打招呼"""
    print(f"你好, {name}!")
```

## 3. Entry Point 约定

Entry point **键**即用户在 CLI 中输入的子命令名：

```toml
[project.entry-points."pyrite.commands"]
ota = "pyrite_ota:app"
mqtt = "pyrite_mqtt:app"
```

对应的 CLI 用法：

```bash
pyrcli ota push COM3 firmware.bin
pyrcli mqtt pub COM3 topic msg
```

命名规则：仅使用小写字母、数字和连字符。

Entry point **值**必须为 `module_path:attribute` 格式，指向一个 `typer.Typer` 实例。

## 4. 安装方式

### 4.1 方式一：pip 安装（推荐）

插件发布到 PyPI 后的标准安装方式：

```bash
pip install pyrite-ota
```

安装后自动可用，无需配置：

```bash
pyrcli ota push COM3 firmware.bin
pyrcli plugin list
# → ota  0.1.0  OTA 无线固件升级
```

### 4.2 方式二：本地插件

供开发中或未发布到 PyPI 的私有插件使用。pyrite-cli 自动扫描两个固定目录：

| 范围 | 目录 | 说明 |
|------|------|------|
| **全局** | `<pyrite-root>/plugin/` | 与 pyrite-cli 安装目录同级，跨项目共享 |
| **局部** | `<CWD>/plugin/` | 项目专属，可提交到版本控制 |

目录结构示例：

```
plugin/
├── ota-tool/
│   ├── __init__.py      # 必须导出 app: typer.Typer
│   ├── commands.py
│   └── requirements.txt # 可选依赖，需手动安装
└── ble-scan/
    └── __init__.py
```

验证本地插件：

```bash
pyrcli plugin list
# → ota-tool   0.1.0  OTA 工具
# → ble-scan   0.1.0  蓝牙扫描
```

### 4.3 方式三：可编辑模式开发

对于按标准包结构组织的插件，开发期间使用 `pip install -e`：

```bash
cd pyrite-ota/
pip install -e .
```

- Python 文件修改即时生效
- `pyproject.toml` 变更后需要重新 `pip install -e .`

这是推荐的开发工作流——兼具自动发现（entry point）和热重载（可编辑安装）。

## 5. 插件元数据

在 `__init__.py` 的模块层级定义以下属性：

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `__plugin_name__` | `str` | entry point key | `pyrcli plugin list` 中显示的名称 |
| `__plugin_version__` | `str` | `"0.0.0"` | `pyrcli plugin info` 中显示的版本号 |

```python
import typer

app = typer.Typer(help="OTA 无线固件升级")

__plugin_name__ = "ota"
__plugin_version__ = "1.2.0"
```

## 6. 访问 pyrite-cli API

插件可以导入并使用 pyrite-cli 的任意模块：

```python
from cli.utils.flash import MicroPython
from cli.utils.webrepl_micropython import WebREPLMicroPython
from cli.utils.transport import Transport
from cli.utils.config import _load_config
from cli.project.sync import ProjectSyncManager
```

> **重要**：pyrite-cli 的内部 API 尚未稳定。请在 `dependencies` 中锁定版本范围：

```toml
[project]
dependencies = ["pyrite-cli>=0.1.0,<0.5.0"]
```

## 7. 发布到 PyPI

```bash
# 1. 构建
pip install build
python -m build

# 2. 上传
pip install twine
twine upload dist/*

# 3. 用户安装
pip install pyrite-ota
```

建议包名以 `pyrite-` 开头便于发现（如 `pyrite-ota`、`pyrite-mqtt`）。

## 8. 完整示例

完整的 `pyrcli greet hello` 和 `pyrcli greet goodbye` 插件：

### 目录结构

```
pyrite-greet/
├── __init__.py
├── commands.py
└── pyproject.toml
```

### `__init__.py`

```python
"""pyrite-greet: pyrite-cli 的问候命令插件。"""

import typer

app = typer.Typer(help="问候命令")

__plugin_name__ = "greet"
__plugin_version__ = "0.1.0"
```

### `commands.py`

```python
import typer
from . import app

@app.command()
def hello(
    name: str = typer.Argument("world", help="要问候的对象"),
    count: int = typer.Option(1, "--count", "-c", help="重复次数"),
):
    """说你好"""
    for i in range(count):
        print(f"你好, {name}! ({i + 1}/{count})")

@app.command()
def goodbye(name: str = typer.Argument(..., help="要道别的对象")):
    """说再见"""
    print(f"再见, {name}!")
```

### `pyproject.toml`

```toml
[project]
name = "pyrite-greet"
version = "0.1.0"
description = "pyrite-cli 的问候命令插件"
requires-python = ">=3.8"
dependencies = ["pyrite-cli>=0.1.0"]

[project.entry-points."pyrite.commands"]
greet = "pyrite_greet:app"
```

### 使用效果

```bash
# 通过 pip 安装
pip install pyrite-greet

# 或放入本地插件目录
mkdir -p plugin/greet && cp -r * plugin/greet/

# 使用
pyrcli greet hello Claude
# → 你好, Claude! (1/1)

pyrcli greet goodbye Claude
# → 再见, Claude!

pyrcli plugin list
# → greet  0.1.0  问候命令
```

## 9. 故障排除

| 现象 | 可能原因 | 解决方法 |
|------|----------|----------|
| `pyrcli plugin list` 不显示插件 | entry point 未注册 | 检查 `pyproject.toml` 中的 `[project.entry-points."pyrite.commands"]` 配置 |
| 运行插件命令时 `ImportError` | 缺少依赖 | pip 插件自动安装依赖；本地插件手动安装：`pip install -r requirements.txt` |
| 本地插件未找到 | 不在正确的目录下 | 将插件放入 `plugin/` 目录（全局：pyrite-cli 同级；局部：CWD 下）。确认 `plugin/<名称>/__init__.py` 存在 |
| 插件命令崩溃 | 插件内部错误 | 查看错误信息；pyrite-cli 用 try/except 包裹每个插件，其他插件不受影响 |
| 可编辑安装后修改未生效 | 需要重新安装 | 修改 `pyproject.toml` 后需重新 `pip install -e .`；代码修改自动生效 |

---

> 三种安装方式选型：生产环境推荐 pip 安装；开发调试使用可编辑模式；私有工具使用本地插件。
