# 条件编译与宏预处理

pyrite-cli 在刷入前对 `.py` 文件进行静态预处理，将 `with feature/target(...)` 块和 `@feature/target(...)` 装饰器展开为 `if True/False:`，由 mpy-cross 的死代码消除在编译时剔除不匹配的代码，减小字节码体积。

---

## 概念

### target vs feature

| 概念 | 语义 | 来源 |
|------|------|------|
| `target` | 硬件平台，互斥 | 设备自动识别 或 `--target` 手动指定 |
| `feature` | 功能特性，可组合 | target 隐式激活 + `--feature` 追加 |

两者在语法上完全相同，均可用作 `with` 块或函数装饰器，区别仅在语义约定。

### active_tags 构成

```
active_tags = resolve_target(自动识别 | --target)
            + target 隐式激活的 features（来自 board_tags 配置）
            + --feature 追加
            - --no-feature 移除
```

## 语法

### `with` 块

```python
with target("ESP32"):
    import network          # 仅 ESP32 编译此块

with target("RP2040"):
    import rp2              # 仅 RP2040 编译此块

with feature("wifi"):
    connect_wifi()          # 仅 wifi tag 激活时编译
```

预处理后展开为：

```python
if True:                    # tag 匹配
    import network

if False:                   # tag 不匹配，mpy-cross 消除
    import rp2
```

行号完全对齐，不插入也不删除任何行。

### 函数装饰器

```python
@target("ESP32")
def connect_wifi():
    ...

@target("RP2040")
def connect_wifi():
    ...
```

匹配的实现保留（装饰器被移除），不匹配的实现包裹在 `if False:` 中：

```python
def connect_wifi():         # ESP32 匹配，装饰器移除
    ...

if False:                   # RP2040 不匹配
    def connect_wifi():
        ...
```

可叠加多个装饰器（逻辑 AND）：

```python
@target("ESP32")
@feature("ble")
def ble_init():
    ...
```

## IDE 类型存根

`pyrcli new` 创建项目时自动生成 `feature_stub.pyi`，让 IDE 不报 `feature`/`target` 未定义错误：

```python
class feature:
    def __init__(self, name: str) -> None: ...
    def __enter__(self) -> "feature": ...
    def __exit__(self, *a: object) -> None: ...
    def __call__(self, fn: object) -> object: ...

target = feature
```

`.pyi` 文件不会被 `flash-program` 收集（仅收集 `.py`），无需手动排除。

## 静态警告分析

预处理时对单文件进行两遍分析，向 stderr 输出黄色警告：

**警告 1：函数所有实现均不匹配**

```
[警告] main.py: 函数 'connect_wifi' 所有实现均不匹配当前 tags
```

触发条件：某函数名的全部 `@target/@feature` 实现均不在 active_tags 中。

**警告 2：裸调用未被 guard 保护**

```
[警告] main.py: 裸调用 'connect_wifi()' 未被 guard 保护，运行时将 NameError
```

触发条件：调用点不在任何 `with target/feature(...)` 块内，且被调函数已被全部禁用。

> 跨文件调用不检测，属于已知限制。

## board_tags 配置

`active_tags` 中 target 隐式激活的 features 由 `board_tags` 映射表决定。

**内置默认值**：

| 关键字 | 激活的 tags |
|--------|------------|
| `ESP32` | `ESP32`, `wifi` |
| `ESP8266` | `ESP8266` |
| `RP2040` | `RP2040` |
| `PICO` | `RP2040` |
| `STM32` | `STM32` |

**在 `pyproject.toml` 中扩展或覆盖**：

```toml
[tool.pyrite.board_tags]
MY_BOARD = ["MY_BOARD", "wifi", "ble"]
ESP32 = ["ESP32", "wifi", "ble"]   # 覆盖默认
```

工具从当前目录向上逐级查找 `pyproject.toml`，找到第一个即停止。

## CLI 参数

### `flash` 和 `flash-program` 新增参数

| 参数 | 说明 |
|------|------|
| `--target TAG` | 手动指定 board target（离线时使用，跳过设备自动识别） |
| `--feature TAG[,TAG]` | 追加激活的 tags，逗号分隔 |
| `--no-feature TAG[,TAG]` | 强制禁用的 tags，逗号分隔 |
| `--manifest PATH` | 指定 manifest.py 路径（仅 `flash-program`） |

**示例**：

```bash
# 自动识别设备 target
pyrcli flash /dev/ttyUSB0 main.py

# 离线指定 target
pyrcli flash /dev/ttyUSB0 main.py --target ESP32

# 追加 feature
pyrcli flash /dev/ttyUSB0 main.py --target ESP32 --feature ble

# 禁用某个 feature
pyrcli flash /dev/ttyUSB0 main.py --target ESP32 --no-feature wifi

# 使用 manifest 刷入目录
pyrcli flash-program /dev/ttyUSB0 ./src --manifest ./manifest.py
```

**无设备时必须指定 `--target`**，否则报错退出：

```
无法识别设备 target，请使用 --target 手动指定
```

## manifest.py

`manifest.py` 控制 `flash-program` 刷入哪些文件，支持按 features 过滤。`pyrcli new` 创建项目时自动生成注释模板。

```python
# manifest.py
module("main.py")
module("lib/utils.py", features=["wifi"])   # 仅 wifi tag 激活时刷入
package("lib/drivers")                      # 递归收集目录下所有 .py
package("lib/net", features=["wifi"])
```

### `module(filename, remote=None, features=None)`

| 参数 | 说明 |
|------|------|
| `filename` | 相对于项目目录的文件路径 |
| `remote` | 设备上的目标路径，默认同 `filename` |
| `features` | tag 列表，任意一个在 active_tags 中即包含；`None` 表示无条件包含 |

### `package(dirname, remote=None, features=None)`

递归收集目录下所有 `.py` 文件，参数同 `module`。

## `preprocessor.py` — 内部 API

```python
from cli.utils.preprocessor import preprocess

result = preprocess(source: str, active_tags: set[str], filename: str = "") -> str
```

- `source` — 原始 Python 源码
- `active_tags` — 当前激活的 tag 集合
- `filename` — 用于警告信息中的文件名（可选）
- 返回展开后的源码字符串

预处理在 `flash_file` 内部自动调用，通常不需要直接使用。
